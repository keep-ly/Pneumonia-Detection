## 3. 数据准备

### 3.1 数据下载与存放
- **来源**：Kaggle [Chest X-Ray Images (Pneumonia)](https://www.kaggle.com/datasets/paultimothymooney/chest-xray-pneumonia)
- **存放路径**：`data/chest_xray/`
- **自动检查**：在运行训练前，脚本需检查 `data/chest_xray/train/` 是否存在且包含图像。若不存在，则提示用户手动下载（因Kaggle需认证，不建议自动化下载）；若存在，则直接使用。

### 3.2 数据划分
- 原数据集已提供 `train/`、`val/`、`test/` 三个文件夹，可直接沿用。
- 若需自定义划分，可读取所有图像路径及标签，按 8:1:1 或 7:1.5:1.5 重新划分，但本计划优先使用官方划分。

### 3.3 预处理与数据增强
- **统一尺寸**：`224×224`（适配 ResNet/EfficientNet 输入）
- **像素归一化**：归一化到 `[0,1]`，并按 ImageNet 均值和标准差标准化（若使用预训练模型）
- **训练集增强**（缓解类别不平衡并提高泛化）：
  - 随机水平翻转（概率 0.5）
  - 随机旋转（±15°）
  - 随机亮度/对比度调整（ColorJitter）
  - 随机缩放裁剪（RandomResizedCrop，可选项）
- **验证集/测试集**：仅缩放 + 中心裁剪 + 标准化，无随机增强。

## 4. 模型设计

### 4.1 基础架构（二选一）
- **选项 A**：自定义轻量CNN（适合快速实验）
- **选项 B（推荐）**：迁移学习，使用 ImageNet 预训练的 ResNet-50 或 EfficientNet-B0，替换最后的全连接层为二分类输出（1个神经元 + Sigmoid）。

### 4.2 输出层与激活函数
- 输出维度：1（二分类概率）
- 激活函数：`Sigmoid`（或使用 `BCEWithLogitsLoss` 时不加激活，直接输出 logits）

### 4.3 损失函数
- 使用 **加权二元交叉熵损失（Weighted BCE Loss）**，对肺炎类别赋予更高权重。
- 权重计算：根据训练集正负样本比例，`pos_weight = 负样本数 / 正样本数`（约 1341/3875 ≈ 0.346，但肺炎是正类，我们希望提高其权重，可设为 `neg/pos * 调节系数`，实际常用 `pos_weight = [neg/pos]`）。在 PyTorch 中可通过 `BCEWithLogitsLoss(pos_weight=torch.tensor([weight]))` 实现。

### 4.4 优化器与学习率
- 优化器：Adam 或 SGD（带动量）
- 学习率：初始 1e-4（微调预训练）或 1e-3（从头训练），并配合余弦退火或 ReduceLROnPlateau 调度。
- 批量大小（Batch Size）：32 或 64（根据显存调整）
- 训练轮数（Epochs）：30~50，早停（Early Stopping）基于验证集损失。

## 5. 训练流程

### 5.1 执行入口
运行 `python src/train.py --config configs/config.yaml`

### 5.2 训练步骤
1. 检查数据目录是否存在，若缺失则报错并提示下载。
2. 加载配置文件，设置随机种子（保证可复现）。
3. 构建训练/验证 DataLoader。
4. 初始化模型（若使用预训练，则冻结除全连接层外的参数，先训练分类头几个 epoch，再解冻微调）。
5. 定义损失函数、优化器、学习率调度器。
6. 循环每个 epoch：
   - 训练阶段：前向传播、计算损失、反向传播、优化器更新，记录训练损失和准确率。
   - 验证阶段：不计算梯度，计算验证损失和准确率。
   - 保存验证损失最低的模型 checkpoint。
   - 输出当前 epoch 的训练/验证指标到控制台和 TensorBoard。
7. 训练结束后，保存最终模型权重到 `outputs/checkpoints/best_model.pth`。

### 5.3 类别不平衡处理补充
- 除加权损失外，可考虑 **Focal Loss** 作为替代方案，但本计划默认使用加权 BCE。
- 数据增强中的过采样（如对肺炎类进行额外增强）可作为可选策略。

## 6. 评估与可视化

### 6.1 评估指标
在 **验证集**（或独立的测试集）上计算：
- Accuracy, Precision, Recall, F1-Score （二分类，肺炎为正类）
- AUC-ROC
- 混淆矩阵

### 6.2 可视化输出
生成并保存以下图像到 `outputs/results/`：
- ROC 曲线（含 AUC 值）
- 混淆矩阵热力图
- （可选）训练/验证损失与准确率曲线

### 6.3 执行评估
运行 `python src/evaluate.py --checkpoint outputs/checkpoints/best_model.pth --data_split val`（也可指定 test 集）。

## 7. 配置管理

所有可调参数统一存放于 `configs/config.yaml`，示例内容：

```yaml
data:
  data_root: "data/chest_xray"
  image_size: 224
  batch_size: 64
  num_workers: 4

model:
  architecture: "resnet50"   # 或 "efficientnet-b0" 或 "custom_cnn"
  pretrained: true
  dropout_rate: 0.2

training:
  epochs: 40
  learning_rate: 1e-4
  weight_decay: 1e-5
  lr_scheduler: "cosine"    # 或 "step"
  early_stopping_patience: 7
  pos_weight: 0.35          # 可根据实际统计计算，或设为 auto 自动计算

augmentation:
  horizontal_flip_prob: 0.5
  rotation_degrees: 15
  brightness_range: [0.8, 1.2]

logging:
  log_dir: "outputs/logs"
  checkpoint_dir: "outputs/checkpoints"
  result_dir: "outputs/results"
```

## 9. 执行与验证步骤

1. **准备数据**：手动下载 Kaggle 数据集，解压到 `data/chest_xray/`。
2. **安装依赖**：`pip install -r requirements.txt`
3. **调整配置**：根据需要修改 `configs/config.yaml`（如 batch size、学习率）。
4. **开始训练**：`python src/train.py`
5. **查看日志**：`tensorboard --logdir outputs/logs`
6. **评估模型**：`python src/evaluate.py --checkpoint outputs/checkpoints/best_model.pth`
7. **产出**：
   - 最佳模型权重 `best_model.pth`
   - 评估图表（ROC、混淆矩阵、损失曲线）
   - 指标数值（打印或保存为 `.txt`）

## 10. 注意事项

- 确保 GPU 可用（若没有，自动切换到 CPU）。
- 训练过程中定期保存中间 checkpoint，防止意外中断。
- 数据增强不要过度，避免破坏医学图像的关键特征（如不要进行垂直翻转）。
- 在验证集上若性能不达预期（如 AUC<0.85），可调整模型规模、增加训练轮数或调高肺炎类权重。
